​ A declarative, efficient, and flexible JavaScript library for building user interfaces

按我的理解就是 react 就是一个专注于 UI 的库

​ 用 react 半年已久，感觉自己始终是个单纯的使用者，而不是从设计者角度来考虑的使用者，文章主要是记录一些对 react 知识的一个梳理以及不断深入的过程。

​ 将会从组件 (Component) 以及组件的属性 (props) 、状态 (state) 、生命周期等方面进行探讨。

React是什么？

​ React 是 Facebook 推出的开源 JavaScript Library，一个用于组建用户界面的JavaScript库，以更简单的方式来创建交互式用户界面，许多革新性的 Web 观念开始流行起来，比如 Virtual DOM、Component，声明式渲染等。

一些特性
  1. 当数据改变时，React 将高效的更新和渲染需要更新的组件，声明式视图使代码更可预测，更容易调试。
  2. 构建封装管理自己的状态的组件，然后将它们组装成复杂的用户界面，由于组件逻辑是用JavaScript编写的，而不是模板，所以可以轻松地通组件传递数据，并保持DOM状态。
安装

通过 Create React App 来完成

npx create-react-app my-app
cd my-app
npm start

ReactDOM.render

ReactDOM.render(element, container,[, callback])

ReactComponent render(
  ReactElement element,
  DOMElement container,
  [function callback]
)

渲染一个ReactElement到DOM中

  • element，要渲染的的 ReactElement
  • container，渲染结果在 DOM 中插入的位置
  • callback，回调函数，可选。传入该值时，会在组件渲染完或更新完成后调用
  • 返回值：React组件
ReactDOM.render(<App />, document.getElementById('root'));

看过一篇关于 ReactDom.render 的文章， APP就是 Component 。而<APP /> 是 Element，更详细的内容请看这里。

HTML 标签 与 React 组件

React 可以渲染 HTML 标签 (strings) 或 React 组件 (classes)， 要渲染 HTML 标签，只需在 JSX 里使用小写字母开头的标签名。

let divElement = <div className="div" />;
React.render(divElement, document.root);

渲染React 组件，只需创建一个大写字母开头的本地变量。

let MyComponent = ...;
let myElement = <MyComponent someProperty={true} />;
React.render(myElement, document.body);
  • React 的 JSX 里约定分别使用首字母大、小写来区分本地组件的类和 HTML 标签。

JavaScript 表达式

属性表达式

使用 JavaScript 表达式作为属性值，使用一对花括号{}包围而不是 ""。

JSX
let person  = <Person name = {user.isLogin ?  user.name: ''} />;
子节点表达式
JSX
let content = <Container>{user.isLogin ? <Nav /> : <Login />}</Container>;
注释

JSX 里添加注释就是在标签的子节点用{}包围。

class ReactDemo extends Component {
  render() {
    return (
      <View style={styles.container}>
        {/*这是一段注释*/}
        <Text style={styles.tips}
          textShadowColor='yellow'>
          React Native!
        </Text>
      </View>
    );
  }
}

延展属性 (Spread Attributes)

let props = {};
props.foo=x;
props.bar=y;
let component = <Component {...props} />;

传入的对象会被复制到组件内。

​ 可以多次使用，也可以和其它属性一起用，但是必须得注意顺序，和所有代码一样，后面会覆盖前面，…是ES6数组里的延展操作符(Spread Operator)，

let props = {foo: 'default};
let component = <CardComponent {...props} foo={'override'} />

组件 (Component)

React 允许将代码封装成组件 (Component)，然后在代码里置入这个组件。

class HelloWorld extends React.Component{
    render() {
        let views=[];
        views.push(<h1>Hello {this.props.name}</h1>);
        return views;
    }
}

ReactDOM.render(<HelloWorld name={'world'}/>, document.getElementById('root'));

哈哈～，真是万物皆可 Hello World～

​ HelloWorld 就是一个组件类，模板置入HelloWorld时，会自动生成 HelloWorld实例，所有组件类都必须有自己的render方法，输出组件，注意：组件类的第一字母必须大写。

属性 (props)

​ 一般都是通过 this.props.xxx的形式获取组件对象的属性，对象的属性可以任意定义，但要避免与 JavaScript 关键字冲突。

遍历对象属性

​ this.props.children 会返回组件内对象的属性，React 提供 React.Children 来处理 this.props.children，我们可以用 React.Children.map或React.Children.forEach 来遍历子节点。

React.Children.map

React.Children.map(children, **function**[(thisArg)])

​ 包在 children 里的每个函数，调用的函数的 this 设置为 thisArg ，如果 children 是一个嵌套的对象或数组，它将被遍历。如果 children 是 null 或 undefined ，返回 null 或 undefined 而不是一个空数组。

React.Children.forEach

React.Children.forEach(children, function[(thisArg)])

class NotesList extends React.Component{
    render(){
        return (
            <ol>
                {
                    React.Children.map(this.props.children,(child)=> {
                        return <h1>{child}</h1>;
                    })
                }
            </ol>
        );
    }
}

ReactDOM.render(<NotesList>
    <span>hello</span>
    <span>world</span>
</NotesList>, document.getElementById('root'));
PropTypes

​ 组件内的属性可以接受任意值，字符串、对象、函数等等都可以，有时我们需要一种机制，验证别人使用组件时，提供的参数是否符合要求。 组件类的 PropTypes 属性，就是用来验证组件实例的属性是否符合要求。

class Wedge extends Component {
    static propTypes = {
      outerRadius: PropTypes.number.isRequired,
    };
    render() {
        return <h1> title:{this.props.outerRadius} </h1>;
    }
}
ReactDOM.render(<Wedge />, document.getElementById('root'));

​ 可以看到上面的 Wedge组件有 outerRadius 属性，PropTypes 告诉 React，outerRadius 属性是 number 类型且必须的，对于非 number 类型，控制台会报出 Warning: Failed propType: Invalid prop outerRadius of type string supplied to Wedge, expected number。

isRequired 只是 PropTypes 其中的一个属性，更多请看官方文档。

默认属性 (defaultProps)

除了既定属性，还可以通过 defaultProps 用来设置组件属性的默认值。

class Wedge extends Component {
    static defaultProps = {
      outerRadius: 100
    };
     render() {
        return <h1> {this.props.outerRadius}</h1>;
    }
}
ReactDOM.render(<Wedge/>, document.getElementById('root'));

上述代码会输出 100。

ref 属性(获取真实的 DOM 节点)

​ 组件并不是真实的 DOM 节点，而是存在于内存之中的一种数据结构，叫做虚拟 DOM (virtual DOM)。只有当它插入文档以后，才会变成真实的 DOM 。根据 React 的设计，所有的 DOM 变动，都先在虚拟 DOM 上发生，然后再将实际发生变动的部分，反映在真实 DOM上，这种算法叫做 DOM diff ，它可以极大提高网页的性能表现。

但是，有时需要从组件获取真实 DOM 的节点，这时就要用到 ref 属性。

class Alert extends React.Component {
    showAlert(message) {
        alert(`Debug:${message}`);
    }

    render() {
        return null;
    }
}

class Demo extends React.Component {
    onClick = () => {
        this.refs.alert.showAlert('Demo');
    };

    render() {
        return <div>
            <h1 onClick={this.onClick}>Click me</h1>
            <Alert ref='alert'/>
        </div>;
    }
}

ReactDOM.render(<Demo/>, document.getElementById('root'));

​ 组件 Demo 的子节点有一个 Alert 组件，为了调用组件提供的方法，必须获取到真实的dom 节点，虚拟 dom 无法获取用户输入，所以必须设置 ref 属性，然后 this.refs[refName]，即可获取真实 dom 节点。

​ 由于 this.refs[refName] 属性获取的是真实 dom ，所以必须等到虚拟 dom 插入文档以后，才能使用这个属性，否则会报错，上面代码中，通过为组件指定 Click 事件的回调函数，确保了只有等到真实dom 发生 Click 事件之后，才会读取 this.refs.[refName] 属性。

React 组件支持很多事件，请查看官方文档。

State

​ 组件会根据 props 的变化进行渲染，但组件无法改变自身 props，为了实现交互，这时候就用到了组件的 state，它是组件私有的，通过 state = {} 初始化，this.setState() 更新，而不是直接在 this.state 上修改。

const clickCount = this.state.clickCount;
this.setState(clickCount: clickCount + 1); 
而不是 this.state.clickCount = clickCount + 1;

​ 主要还是因为 this.state 是一个对象，去改变对象的值是毫无意义的，React 中只有去驱动UI的更新才会有意义，虽然可以尝试直接改变 this.state，但并没有驱动 UI 的重新渲染，这种操作也就毫无意义，所以需要使用this.setState 来驱动组件的更新过程。

​ setState() 更新组件状态之后不会立即生效，react 为提高性能会按批次更新 state 然后 render, 即异步操作，所以 setState() 之后立即去取 state 的值并不是更新之后的状态。

setState() 第一个参数接受两种类型的参数，Object 以及Function

this.setState({
    msg: '更新msg'
});

参数为 Object 时，可以即为对应 state 中的 key, value 即是新的值。

参数为 Function 时，setState()会将上一个结果作为参数传入这个函数

this.setState(prevState => { counter: prevState.counter + 1 })

setState() 第二个参数是回调函数，表示 state 更新完成。

this.setState({
    msg: '更新msg'
}, () => {
    console.log('更新完毕')
});

可以使用 Promise 以及async/await,封装成同步/异步操作。
